# HWP Editor Analysis

## 목표

Bomcare Suite 내부 편집기를 단순 HTML 편집기에서 `표준 HWP 작업 흐름을 이해하는 문서 편집기`로 확장한다.

## 기준으로 본 핵심 기능

한컴 도움말과 한컴독스 지원 문서를 기준으로 보면, HWP 편집기의 핵심은 아래 5개 축으로 정리된다.

1. 문서 구조
- 스타일/문단 모양/글자 모양
- 쪽 설정(용지 방향, 여백, 머리말, 꼬리말)
- 쪽 번호, 쪽 나누기, 구역 개념

2. 문서 객체
- 표 생성, 셀 편집, 병합/분할
- 이미지, 도형, 결재란, 캡션

3. 검토/협업
- 메모
- 변경 내용 추적
- 버전 비교

4. 파일 형식
- HWP/HWPX 읽기/쓰기
- HWPX는 단순 텍스트가 아니라 구조화된 문서 포맷 처리 필요

5. 운영 기능
- 자동저장
- 버전 목록
- 인쇄/PDF/HWPX 내보내기

## 현재 구현 상태

이번 단계에서 반영:
- 저장본 목록
- 자동저장
- 문서 스타일 프리셋
- 쪽 설정(세로/가로, 여백, 머리말/꼬리말 간격)
- 머리말/꼬리말
- 쪽 번호 토글
- 표 삽입
- 결재란 삽입
- 쪽 나누기

아직 미구현:
- 셀 병합/분할, 표 속성 상세 편집
- 이미지/도형/캡션
- 메모/변경 추적
- 실제 OWPML 기반 HWPX 패키지 생성
- 바이너리 HWP 호환

## 현실적인 로드맵

### 1단계
- 현재처럼 내부 웹 편집기 고도화
- 운영일지/회의록/결재서 양식별 템플릿 강화

### 2단계
- 검토 기능 추가
- 메모, 변경 내용 표시, 승인 전 비교 화면

### 3단계
- 실제 문서 변환 파이프라인
- HWPX 패키지 생성 또는 외부 변환 엔진/SDK 연동
